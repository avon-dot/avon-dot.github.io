MySQL服务Logs

发表于 2023-03-19 更新于 2023-03-20 分类于 数据库

MySQL服务有多种日志，可以帮助使用者找出那些活动的发生。

日志类型 描述
错误日志 Error log mysqld 开始，运行或者停止的时候发生的问题日志
一般查询日志 General query log 当客户端连接、断开连接、接收到客户端的SQL语句时，会向general log中写入日志
二进制日志 Binary log 更改数据的语句，包括复制的服务器
延迟日志relay log 从复制源服务器接收到的数据更改
慢查询日志Slow query log 超过long_query_time(最大的查询时间，默认10s)的查询语句
DDL log（metadata log） 数据定义语句对元数据的操作记录

高性能MySQL阅读笔记-Schema与数据优化

发表于 2023-03-04 更新于 2023-03-05 分类于 读书笔记

选择优化的数据类型

正确的数据类型选择原则

  • 更小的通常更好
    • eg.只需要存储0-20，tinyint unsighned更好
    • 需要确保没有低估存储值的范围，增加数据类型的范围是一个耗时且痛苦的操作
  • 简单就好
    • 整型比字符操作代价更小
    • 应该使用MySQL的内建类型date、time、datetime来存储日期和时间
    • 应该使用整型来存储IP地址
  • 尽量避免NULL
    • 通常情况下指定维NOT NULL，除非真的需要存储NULL
    • 查询中如果包含有NULL的列，可为NULL的列使得索引、索引统计和值的比较更复杂
    • 可为NULL的列会使用更多的存储空间，为索引的列尽量避免成为可为NULL的列
    • 例外：InnoDB使用单独的bit存储NULL值，所以对于稀疏矩阵有很好的空间效率，但并不是适用于MyISAM
